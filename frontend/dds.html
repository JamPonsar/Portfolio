Dolphin Data Sharing

Decentralized File-Sharing Platform with Blockchain Integration

September 2024 - December 2024

View on GitHub Live Demo

Project Overview

Dolphin Data Sharing is a sophisticated decentralized file-sharing application that leverages blockchain technology to provide secure, peer-to-peer file distribution. Built during my time at Stony Brook University, this project demonstrates advanced concepts in distributed systems, cryptography, and modern web development.

Key Features

Cryptographic Security

Hash-based file uploads and downloads ensuring data integrity and security

P2P Functionality

Peer-to-peer proxy functionality for efficient file distribution

Metadata Storage

MongoDB integration for efficient file metadata management

Blockchain Integration

Bitcoin tools (btcd, btcwallet) for secure transaction handling

Secure Authentication

Wallet creation and hashed password authentication system

Modern UI

Interactive Material UI interface with dark mode support

Technical Implementation

Frontend Technologies

  • React.js: Component-based user interface
  • Material-UI: Modern, responsive design system
  • JavaScript/JSX: Interactive functionality

Backend & Blockchain

  • Go: High-performance backend services
  • btcd: Bitcoin daemon for blockchain operations
  • btcwallet: Wallet management and transactions
  • DHT: Distributed hash table for P2P networking

Data & Security

  • MongoDB: File metadata and user data storage
  • Cryptographic Hashing: File integrity verification
  • HTTP Proxy: Secure file access management

Challenges & Solutions

Blockchain Integration Complexity

Integrating Bitcoin tools with a modern web application required extensive research and custom Go libraries to bridge the gap between blockchain operations and user-friendly interfaces.

P2P Network Reliability

Ensuring consistent peer discovery and file availability across the distributed network involved implementing robust DHT mechanisms and fallback strategies.

Security Implementation

Balancing user accessibility with cryptographic security required careful design of the authentication system and secure key management.
